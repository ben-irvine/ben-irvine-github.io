An analogy to describe HTML and CSS differences

Creating a website with HTML and CSS can be compared to building a house. Writing the HTML code is making the blueprint of the house, while the CSS code is the interior decoration designs. When all is ready, the plans are handed over to a builder/decorator named Browser, who throws the whole thing together in a matter of seconds.



Explain control flow and loops using an example process from everyday life

What is control flow in Javascript and why is it important to know about as a Web Developer. Writing Javascript is essentially writing a set of instructions for the computer to do and controlling the flow is controlling the order and pathway of operations, the two handiest tools for this being if/else statements, and loops. What these tools do is check certain criteria and act differently based on whether these criteria are met.


When crossing the road, I imagine you follow a flow of actions to make sure you get to the other side safely. First you check one way to see if it's clear, then the other. If it’s not clear, you act accordingly, waiting for the car or cars to pass by. At this point you perform your checks again and follow this flow until both ways are clear. When this finally happens, you cross the street safely and the flow of events for crossing the street comes to an end.



This is a simple if/else loop. The action you perform depends on the set criteria, but it loops into you repeating the criteria check until you can perform the desired action.



Describe what the DOM is and an example of how you might interact with it

The DOM is an interface within a web browser through which we can view webpages as tree structures, made up of objects linking to their parent structures all the way back to a starting point. It is useful because it has a range of tools that can be used to edit or experiment on web pages, including a console that allows us to interact with the webpage using Javascript commands, allowing us to alter the pages structure, style and contents in response to certain 'events'.


Amongst these tools are a range of search functions, commands and listeners (code that listens for and reacts to certain things happening like clicks or button pushes). With these commands an incredible range of unique features can be created to make a web page interactive and the DOM is a convenient window into experimentation and debugging these features.

Explain what functions are and why they are useful

In Javascript, a function is a sequence of code that is assigned to a chosen keyword that can be called on freely after being declared. ‘Declaring’ a function is simply stating the name of the function and writing out the code that the function will execute. Once declared, a function can be executed by simply stating its name.


Functions are incredibly useful as web developers and programmers frequently need to repeat various sequences of code. Once declared, functions are a great time saving shortcut for programmers, saving them from rewriting or copy and pasting their previous code sequences.
