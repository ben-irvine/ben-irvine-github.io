CSS - Inline Display or Inline-Block Display

In CSS, the 'display' property influences how an element is shown on the page. The values 'inline' and 'inline-block' both place the element on the same horizontal line as the element before itself (assuming there is space for it), so what is the difference?


Default Display Values

Starting with the basics of display:

Elements with an inline display only take up as much space as needed for their content and their size cannot be altered. Inline elements can sit next to each other on the horizontal line or slot into other elements comfortably. Inline elements are useful for putting small additions into block level elements e.g. using an anchor tag to put a link into a block level paragraph. Default inline elements include a (anchor), span and img. The code to the right shows an anchor element used inside a block paragraph element.

Elements with a block display value possess an inherent line break, meaning they always start on a new line and always take up the whole line. They can also have their height and width adjusted to better suit the layout of the page or the content of the element. Default block elements include div, p, h1-h6, header, footer and section. Inline-Block Elements

Web developers can change the display of elements in CSS to change how to are shown. Giving an element an inline-block display value essentially turns it into a block element without the aforementioned inherent line break. This means its height and width can be altered and it can sit on the same row as other inline or inline-block elements.
